欢迎来到 Git 的世界！作为代码版本控制的核心工具，Git 就像是为你项目打造的“时间机器”**和**“游戏存档系统”。只要你掌握了它的核心逻辑，后面协作和管理代码就会变得非常轻松。

既然你已经安装好了 Git，那我们直接从最核心的概念和最常用的命令开始，手把手带你通关。

---

### 必备核心概念：Git 的“三个区域”

在正式敲命令之前，一定要理解 Git 里的三个核心区域，这是所有操作的基础：

1. **工作区 (Working Directory)**：你在电脑里能直接看到的文件夹。你在这里新建、修改或删除文件。
2. **暂存区 (Staging Area)**：一个临时准备区（舞台）。你把改好的文件放到这里，意思是“这些文件我准备好要存档了”。
3. **版本库 (Local Repository)**：Git 帮你保存历史记录的“保险箱”。一旦提交到这里，你的代码就成功存了一个“历史挡位”。

---

### 第一步：配置你的“名片”（仅需设置一次）

Git 需要知道是谁在修改代码。打开你的终端（Windows 用户搜索打开 `Git Bash`，Mac 用户打开 `Terminal`），输入以下命令配置你的用户名和邮箱：

```bash
git config --global user.name "yunfan"
git config --global user.email "3240906818@qq.com"

```

> **提示**：这里的名字和邮箱不需要是真实的，但在团队协作中，最好用别人能认出你的名字。

---

### 第二步：本地“单机”最简流程

现在我们在本地创建一个项目，并完成你的第一次“代码存档”。

#### 1. 初始化仓库

在电脑里新建一个空文件夹（比如叫 `my-project`），进入这个文件夹，在终端里运行：

```bash
git init

```

*这会在文件夹里生成一个隐藏的 `.git` 文件夹，代表这个目录正式被 Git 管理了。*

#### 2. 查看当前状态

在整个开发过程中，这个命令你会敲无数次：

```bash
git status

```

*它会告诉你现在哪些文件被修改了、哪些放到了暂存区。*

#### 3. 把文件搬上“舞台”（添加到暂存区）

我们在文件夹里新建一个叫 `readme.md` 的文件。此时运行 `git status` 会看到文件显示为红色（未追踪）。
把文件添加到暂存区：

```bash
git add readme.md

```

*如果你修改了很多文件，想一次性全部添加，可以使用：* `git add .`

#### 4. 正式存档（提交到版本库）

文件进入暂存区后（输入 `git status` 会变成绿色），就可以正式打包存档了：

```bash
git commit -m "这是我的第一次提交，新增了readme文件"

```

* `-m` 后面的字符串是**提交说明（Commit Message）**，一定要写清楚你这次改了什么，方便以后回溯。*

#### 5. 查看存档历史

想看看自己过去都存了哪些挡？

```bash
git log

```

*这里会列出每一次提交的作者、时间和提交说明。*

---

### 总结：本地开发常用“四部曲”

其实，你日常在本地开发时，90% 的时间都在循环这四个步骤：

1. **写代码**（修改文件）
2. `git status`（查看哪些东西变了）
3. `git add .`（把改动全部拉到暂存区）
4. `git commit -m "完成了XX功能"`（打包存档）

---

### 第三步：连通远程仓库（以 GitHub / Gitee 为例）

代码总存在本地容易丢失，我们需要把它同步到云端（比如 GitHub 或者是国内的 Gitee）。

1. **在云端新建仓库**：登录 GitHub/Gitee，点击右上角 `+` 号 -> `New repository`。新建一个干净的仓库，**不要**勾选“Initialize this repository with a README”（因为我们本地已经有了）。
2. **关联远程仓库**：复制云端仓库给你的 `HTTPS` 链接，在本地终端运行：
```bash
git remote add origin <你复制的仓库链接>

```


3. **推送到云端**：将本地的代码推送到远程仓库的主分支（现在通常默认叫 `main` 或 `master`）：
```bash
git push -u origin main

```


*(加上 `-u` 参数后，以后你只需要简单地输入 `git push`，Git 就会自动推送到这个对应的远程分支了。)*

### 避坑与新手常见疑问

* **Q：我不小心写错代码了，怎么回到上一个存档？**
* **答**：如果只是想放弃工作区里还没来得及 `add` 的修改，可以使用 `git checkout -- <文件名>`（新版 Git 也可以用 `git restore <文件名>`）。如果已经 commit 了想彻底回退，可以使用 `git reset --hard HEAD^`（注意：这会抹去你最新的改动，请谨慎使用）。


* **Q：`git add` 之后，我还能修改文件吗？**
* **答**：可以。但修改后，你需要再次运行 `git add` 把最新的修改放进暂存区，否则 commit 的时候只会保存你上一次 `add` 时的状态。


* **Q：怎么把别人的代码下载到本地？**
* **答**：使用 `git clone <仓库链接>`，这会直接把远程的整个项目完整地复制到你本地。
